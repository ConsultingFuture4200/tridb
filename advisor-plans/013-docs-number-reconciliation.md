## Why this matters

TriDB is being prepared for an open-source launch whose GTM doc itself says external credibility
is make-or-break. Right now the single most-quoted number (the SM-2 latency ratio) appears as
**four different values** across the four public-facing docs, and the GTM doc attributes a
2k-entity/dim-32/x86 measurement to "100k/dim-768, GX10" — precisely the toy-scale attack the doc
tries to preempt. Several status docs also contradict shipped reality (STATUS/ADR-0012 say
`tjs_open(B)` is unbuilt while README + git say it shipped; ADR-0011 says degree stats don't
exist while `join_order_legstats.c` reads them; the spec lists resolved markers as open; the v0
extension README calls the built v1 AM a "never compiled skeleton"). A hostile launch reader
cross-checking any two of these finds a contradiction in minutes.
