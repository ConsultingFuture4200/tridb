# TriDB Benchmark — Filtered Vector Search (VectorDBBench IntFilter methodology)

**Fused filter+vector with early termination on REAL SIFT-128.** Recall@10 and latency vs filter SELECTIVITY (`WHERE label >= t ORDER BY emb <-> q LIMIT k`), on 1000000 base vectors, 100 queries, median of 5 warm runs.

| selectivity (% pass) | recall@k | median latency (ms) | QPS / conn |
|---:|---:|---:|---:|
| 1% | 1.000 | 40.114 | 25 |
| 5% | 1.000 | 42.556 | 23 |
| 10% | 1.000 | 45.480 | 22 |
| 50% | 1.000 | 66.279 | 15 |
| 99% | 1.000 | 87.898 | 11 |

## Notes

- **Selectivity = % of rows the filter PASSES** (1% is the hard VDBB case: the predicate excludes 99% of the corpus). Recall is graded vs the EXACT filtered top-k numpy oracle on the same labels (deterministic from the seed).
- **Dataset:** sift-128-euclidean (real). Single warm connection; QPS/conn = 1000/median-latency (concurrent QPS is the VDBB headline — run the recognized tool via bench/vdbb_tridb.py for the multi-client number).
- **Scale:** this is the bounded slice for the standin; the full SIFT-1M and the VDBB 768D1M1P/99P Cohere cases run on the GX10 (NEON HNSW).
