# TriDB fused (graph+vector) link recovery vs the cosine-only lower bound (v0.1.0)

> **Revision note (reviewer findings applied).** This version adds a leave-out-the-positives
> leakage control, a popularity/degree baseline, paired significance (bootstrap CI + Wilcoxon),
> and a cosine-vs-Adamic-Adar orthogonality measurement, and reframes the metric as **link
> recovery / reconstruction proxy** rather than held-out prediction. The headline lift shrank
> after the leakage correction (raw +21% → corrected +13.3%) and several earlier claims were
> corrected — see the callouts below.

**What this is.** The cheapest direct test of TriDB's core thesis on the real full-Wikipedia
graph (DEV-1354): given a candidate pool generated by embedding similarity, does **graph
topology re-rank the real hyperlinks higher than cosine alone?** If yes, topology carries
signal beyond the embedding; if fused ≈ cosine, the thesis gets no support here.
**This is a link-RECOVERY / reranking test (a reconstruction proxy), NOT a latency claim and
NOT held-out link prediction** (ADR-0017: the I/O-bound regime is expected-fail; no speed win
is claimed). The positives are *existing* out-edges and overlap@k measures the fraction of the
top-k rerank that is already a link — i.e. precision@k on a cosine pool, not recovery of a
held-out edge set.

TL;DR
- **Topology adds a real but small signal that survives a leakage correction.** On the identical
  50-candidate cosine pool, **leakage-corrected** Adamic-Adar reranking lifts overlap@10 from the
  **0.110 cosine lower bound to 0.1246 (+13.3% relative, paired 95% CI [+0.0118, +0.0176],
  Wilcoxon p≈2.6e-24)**, and RRF-fused cosine+AA reaches **0.1261 (+14.6%)**. The lift is
  statistically significant, not noise — but it is a nudge, not a step-change (ceiling ≈ 0.21).
- **~36% of the raw lift was a reconstruction artifact.** Without the control, AA scored 0.1328
  (+21%); removing A's *other pooled true out-edges* from its successor set before scoring drops
  it to 0.1246. The gap (+0.0082) is inflation from interlinked pooled positives, not signal.
- **It is not a popularity artifact.** A degree-only reranker (order the pool by candidate
  in-degree) scores **0.0955 — below cosine and well below AA** — so the lift is attributable to
  structure, not to a popularity prior.
- **Fusion is (narrowly) the best method after the correction.** With the corrected AA, RRF-fused
  (0.1261) edges AA-alone (0.1246). The earlier "AA-alone beats fusion" finding was itself a
  leakage artifact: it held only on the *uncorrected* AA (0.1328).
- **Cosine and AA are only weakly rank-correlated (Spearman 0.17).** Within a pool, AA is not
  merely a sharper cosine — it reorders substantially differently and surfaces ~0.50 true links
  per source in its top-10 that cosine misses (vs 0.35 the other way). This is measured evidence
  that topology is a *partly* orthogonal ranker here, though both unique-recovery counts are small.
- **node2vec: skipped by design** (training-based, multi-hour on 6.9M nodes / 224M edges).
- **Scale is real, not a slice:** 6,900,039 embeddings, one streaming pass over the full
  224,475,283-edge graph (221.1M valid after endpoint-membership filter), on the Spark GB10.

## Method

**One canonical protocol, six columns.** Candidate generation is bounded to each source's
cosine **top-50** neighbours from the cuVS CAGRA index (the same GPU path as the baseline).
Every column then scores overlap@10 = *fraction of the top-10 reranked candidates that are
already an out-edge of the source* — the same metric as the 0.110 lower bound. This is **link
recovery / precision@10 on the cosine pool, a reconstruction proxy — not held-out prediction**.
Because the pool and metric are identical for all columns, **cosine-only reproduces ~0.110 as a
built-in sanity check** and the rest are pure reranks of the same pool. This respects TR-1 /
early termination: topology never scores all O(N²) pairs, it only reorders a bounded pool, and
the adjacency materialized is bounded to the ~246k source+candidate nodes, not the full 6.9M.

**Leakage control (leave-out-the-positives).** CN/AA are computed on the full graph that still
contains the very edges being scored. The specific A→B bit is never read, but A's *other* pooled
true out-edges would otherwise act as shared neighbours and inflate CN/AA on exactly the positive
candidates (dense in topical clusters). So before scoring source A we **remove A's pool-
intersecting out-edges (the test positives) from succ(A)**, and report both the corrected number
(honest) and the uncorrected "leaky" number (the gap is the reconstruction inflation). The
label/`linked` flag still uses the full succ(A); only the topological *features* are held out.

**Topological scorers** (no training, computed from adjacency), predicting a directed link A→B:

| Scorer | Definition | Notes |
|---|---|---|
| Common Neighbours | `CN(A,B) = |succ(A) ∩ succ(B)|` | successor/**bibliographic-coupling** variant |
| Adamic-Adar | `AA(A,B) = Σ_{z ∈ succ(A)∩succ(B)} 1/log(indeg z)` | rare shared targets weigh more |

We use **successor sets** (out-neighbours) on both endpoints — bounded by out-degree (mean 31.5),
so intersections stay cheap on 224M edges; the undirected variant would pull in million-degree hub
*in*-neighbourhoods and blow the compute budget.

**Controls & statistics.**
- **Popularity/degree baseline:** rank the pool by candidate global in-degree `indeg(B)`. If a
  degree prior matches AA, the "topology" claim collapses to popularity. (It does not — see below.)
- **Significance:** paired bootstrap (4000 resamples) 95% CI **and** Wilcoxon signed-rank over the
  5000 per-source `(AA_corrected − cosine)` overlap@10 deltas. The design is paired (identical
  sources, identical pool), so a paired test is both easy and powerful.
- **Orthogonality:** per-pool Spearman rank-correlation of the cosine vs AA orderings, plus a
  link-recovery cross-tab (how many true out-edges each ranker's top-10 surfaces that the other
  misses). Low correlation + nonzero unique recovery ⇒ topology is a partly different signal, not
  just a sharper restatement of the embedding.

**Fusion:** reciprocal-rank fusion (RRF, `rrf_k=60`) of the cosine rank and the **leakage-corrected**
Adamic-Adar rank within the pool: `score(B) = 1/(60+rank_cos) + 1/(60+rank_aa)`.

**node2vec (skipped).** Training node2vec on 6.9M nodes / 224M edges means a multi-billion-token
random-walk corpus plus a skip-gram training pass — a multi-hour-to-day job, out of scope for the
cheapest-test remit. Reported as `node2vec_included=false` in the JSON, not silently omitted.

**Honest boundaries.** These embeddings are **dim-384 float32** (bge-small), **NOT** the spec's
dim-768 float8 I/O-pressure basis — this is the compute-bound-ish regime, not the I/O regime, so
no latency conclusion is drawn. Counts are reconciled to the real corpus: **6,900,039 articles /
224,475,283 edges** (the extractor manifest bug lost shards 28/49/71; we trust the rebuilt shard
files — 221,135,287 edges survive the both-endpoints-are-articles filter, ~1.5% dropped).

## Results

Real enwiki, 5000 sampled sources (seed 42), cosine top-50 pool, overlap@10, on the Spark GB10.
Mean out-degree 31.5. **Pool base rate 0.042** — only ~2.1 of the 50 pooled candidates are true
out-edges on average, so a random rerank scores ~0.042 and the perfect-rerank ceiling is
~0.21 (2.1/10). All rankers are measured against that floor and ceiling.

| Ranker | overlap@10 | vs cosine | vs random-in-pool (0.042) |
|---|---:|---:|---:|
| popularity (in-degree prior) | 0.0955 | −0.0145 (−13%) | 2.3× |
| **cosine-only** (= baseline lower bound) | **0.110** | — | 2.6× |
| common-neighbor only *(corrected)* | 0.1222 | +0.0122 (+11%) | 2.9× |
| **adamic-adar only *(corrected)*** | **0.1246** | **+0.0146 (+13.3%)** | 3.0× |
| **fused (RRF cosine+AA)** | **0.1261** | **+0.0161 (+14.6%)** | 3.0× |

**Leakage correction (leave-out-the-positives).** Uncorrected AA scores 0.1328 (+21%); corrected
AA scores 0.1246 (+13.3%). The **+0.0082 gap ≈ 36% of the raw lift was reconstruction inflation**
from interlinked pooled positives — real, and now removed. (Leaky CN 0.1275 → corrected 0.1222.)

**Significance (paired, corrected AA vs cosine).** mean Δ = **+0.0146**, bootstrap **95% CI
[+0.0118, +0.0176]** (excludes 0), **Wilcoxon signed-rank p ≈ 2.6e-24**. The modest lift is real,
not sampling noise.

**Popularity control.** The degree-only reranker scores **0.0955**, *below* cosine — so the AA
lift is not a popularity proxy; structure earns the signal.

**Orthogonality.** Spearman(cosine, AA) = **0.172** over 4,775 pools with a defined AA order —
low. AA surfaces **0.4984** true links per source in its top-10 that cosine misses; cosine
surfaces **0.352** that AA misses. So within the pool the two rankers substantially disagree and
each recovers links the other buries — evidence that topology is a *partly orthogonal* signal
here, not merely a sharper cosine (though both unique-recovery counts are well under one link).

**Sanity check.** cosine-only = **0.10996** (unrounded) vs the standalone baseline **0.1101** — a
**−0.00014 residual**, not bit-identical: it comes from CAGRA approximate-kNN nondeterminism in
pool construction, not a protocol difference. Close enough to confirm the apples-to-apples rerank.

Run cost (GB10): CAGRA build over 6.9M vectors 50.0s; one streaming pass over all 72 edge shards
(224.5M rows) 29.3s; scoring 5000×50 intersections (×2 for the leakage control) negligible.
End-to-end a few minutes — cheap, exactly as intended.

## Verdict

**Topology adds signal — modestly, significantly, and not from popularity. (yes, small)**

After a leave-out-the-positives leakage correction, graph structure still concentrates the true
hyperlinks measurably higher than embedding proximity alone: leakage-corrected Adamic-Adar lifts
overlap@10 by **+0.0146 absolute (+13.3% relative)** over the 0.110 cosine lower bound, and the
paired 95% CI [+0.0118, +0.0176] excludes zero (Wilcoxon p ≈ 2.6e-24). The lift is **not** a
popularity artifact — a degree-only reranker scores 0.0955, below cosine. But it is a nudge, not
a step-change: absolute overlap moves from 0.110 to ~0.125 against a ~0.21 ceiling, and roughly a
third of the *uncorrected* headline (+21%) was reconstruction inflation.

On the corrected numbers **the RRF fusion (0.1261) is narrowly the best method**, edging AA-alone
(0.1246); the earlier "AA-alone beats fusion" observation held only on the leaky AA and does not
survive the control. Cosine and AA are only weakly rank-correlated within a pool (Spearman 0.17)
and each recovers links the other misses, so on this evidence topology behaves as a **partly
distinct ranker of the same topical-proximity landscape** rather than either a redundant copy of
the embedding or a fully independent modality. That is qualified support for the tri-modal thesis:
topology carries some predictive signal the vector index does not, small in absolute terms because
most of a Wikipedia article's global nearest embedding-neighbours are simply unlinked topical peers.

**Scope caveats (do not over-read):** this is a bounded-pool 1-hop **link-recovery / reconstruction
proxy** on dim-384 f32 embeddings, not held-out link prediction and not a latency result (ADR-0017
I/O regime is expected-fail and untested here). Whether the production `tjs_open` join — which can
traverse *multi-hop* topology rather than only 1-hop successor overlap — extracts more than this
1-hop proxy is a **hypothesis this experiment motivates but does not measure**; it is not a proven
lower bound.

## Reproduce

```bash
# On the Spark GB10 (GPU CAGRA + 224M-edge pass), from the repo root.
# --json-out defaults to bench/results/, so this regenerates the committed artifact:
.venv/bin/python tools/wiki_linkpredict_fused.py \
    --corpus data/wiki/enwiki --emb-in data/wiki/enwiki/emb \
    --sample 5000 --seed 42 --pool 50 --k 10 \
    --json-out bench/results/wiki_linkpred_fused.json
```

- Tool: `tools/wiki_linkpredict_fused.py`
- Results: `bench/results/wiki_linkpred_fused.json`
