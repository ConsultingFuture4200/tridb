## Findings considered and rejected (do not re-audit)

- **graph_store.c SPI error path "skips SPI_finish"**: not a leak. PostgreSQL's
  `AtEOXact_SPI` cleans up the SPI stack on the transaction abort that `elog(ERROR)`
  triggers. By-design platform behavior.
- **Dev-only credentials in baseline/docker-compose.yml** (neo4j/testpassword, postgres,
  minioadmin): intentional local test fixtures, explicitly commented "Dev-only … override
  for anything non-local", no real secret value. An `.env.example` is a nice-to-have only;
  not worth a standalone plan.
- **add_edge unbounded array growth / global visit counter / FR-7-is-principle-only /
  emission-level early termination / FROM-clause SRF materialization / scalar-distance-is-0**:
  all already documented as by-design v0 limitations in `docs/graph_store_v0_limitations.md`
  and `docs/fork_findings.md`. Settled, not findings. (Note: 005 *fixes* the scalar-distance
  bug, which the docs frame as a candidate fork patch — that's the action, not a re-report.)
- **C-extension boundary tests (empty adjacency, non-existent vertex, negative paths)**: the
  code is already correct (palloc0); these are nice characterization tests. Folded into 001's
  test expansion if desired; not a standalone plan.
- **Streaming graph-predicate rewrite inside this batch**: the audit found that TJS currently
  precomputes a reachable-id set. Plan 011 fixes measurement honesty first. A full streaming
  redesign is intentionally deferred because it changes the operator architecture and SPI risk
  profile.
